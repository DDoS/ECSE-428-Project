# Gherkin Acceptance Test for Searching Questions

Feature: Search for a Question
	Given I am an anonymous user or registered user
	And I want to find a particular question or a question that contains specific keywords
	I should be able to search for that question by viewing a list of questions matching particular keywords

Scenario: Obtain question search results using one keyword
	Given I open the url "http://mayhem-ecse428.rhcloud.com/questions/search"
	And the title is "Search Questions - Mayhem"
	When I set "keyword" to the input field "#search"
	And I click on the button ".btn.btn-search"
	Then I expect that the title is "Question Search Result - Mayhem"
	And I expect that element "#view_questions_date_asc" becomes visible

Scenario: Obtain question search results using multiple keyword
	Given I open the url "http://mayhem-ecse428.rhcloud.com/questions/search"
	And the title is "Search Questions - Mayhem"
	When I set "keyword1 keyword2 keyword3" to the input field "#search"
	And I click on the button ".btn.btn-search"
	Then I expect that the title is "Question Search Result - Mayhem"
	And I expect that element "#view_questions_date_asc" becomes visible

Scenario: Obtain question search results failed due to empty keyword field
	Given I open the url "http://mayhem-ecse428.rhcloud.com/questions/search"
	And the title is "Search Questions - Mayhem"
	When I click on the button ".btn.btn-search"
	Then I expect that the title is "Question Search Result - Mayhem"
	And I expect that element "#empty" becomes visible
	And I expect that element ".alert.alert-noKeyword" becomes visible

Scenario: Obtain question search results failed due to no matches
	Given I open the url "http://mayhem-ecse428.rhcloud.com/questions/search"
	And the title is "Search Questions - Mayhem"
	When I set "invalidKeyword" to the input field "#search"
	And I click on the button ".btn.btn-search"
	Then I expect that the title is "Question Search Result - Mayhem"
	And I expect that element "#empty" becomes visible
	And I expect that element ".alert.alert-noMatch" becomes visible
